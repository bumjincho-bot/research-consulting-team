# secrets/ — 자격증명 보관소

> ⚠️ **이 폴더는 Git 에 절대 커밋되지 않습니다.**
> 루트 `.gitignore` 와 `secrets/.gitignore` 가 이중으로 차단합니다.

---

## 무엇을 두는가

- `.env` — 유료 사이트 ID/PW/API key 실제 값 (`/.env.example` 형식)
- `*.key`, `*.pem`, `*.json` — 인증서, 서비스 계정 키 (있을 경우)
- 그 외 자격증명 관련 파일

## 무엇을 두지 않는가

- 보고서 출력물
- 리서치 데이터
- 캐시
- 코드

이 폴더는 **자격증명만** 둡니다. 다른 산출물은 다른 폴더로.

---

## 신규 멤버 온보딩

1. **이 스킬 폴더 통째로 받기** (zip / 팀 클라우드 드라이브 / 직접 전달)
2. 루트에서:
   ```bash
   cp .env.example secrets/.env
   ```
3. 팀 공유 채널 (1Password vault / Bitwarden / 사내 비밀 관리 도구) 에서 실제 값 받아 `secrets/.env` 채우기
4. 셸에 로드:
   ```bash
   set -a
   source secrets/.env
   set +a
   ```
5. 검증:
   ```bash
   echo "${STATISTA_API_KEY:0:4}***"   # 처음 4자만 표시 — 전체 값 출력 금지
   ```
6. `policies/credentials-policy.md` 와 `policies/env-loading.md` 정독

## 자격증명 로테이션 일정

| 항목 | 주기 | 책임자 |
|---|---|---|
| 모든 유료 사이트 비밀번호 | 분기 1회 (1/4/7/10월 첫 주) | (팀에서 지정) |
| API key | 분기 1회 또는 누출 의심 시 즉시 | (팀에서 지정) |
| 멤버 퇴장 시 | **즉시 전수 로테이션** | (팀에서 지정) |
| 사고 발생 시 | **즉시 + 사고 보고** | (팀에서 지정) |

---

## 사고 대응 절차 (자격증명 유출 의심)

1. **즉시** 해당 사이트 로그인 → 비밀번호 변경 + API key 폐기·재발급
2. 활성 세션 전부 종료 (가능한 경우)
3. 최근 로그인 이력·API 호출 이력 확인
4. 팀에 즉시 공유 (어떤 자격증명, 언제, 어떻게 노출됐는지)
5. `secrets/.env` 의 해당 값 새 값으로 교체
6. learning-log/runs/ 검사 → 평문 노출 흔적 있으면 해당 run 삭제 후 페르소나·도구 사용 패턴 점검

## 절대 금지

- ❌ `secrets/.env` 또는 그 값을 Slack/Notion/이메일에 평문 첨부
- ❌ AI 어시스턴트 프롬프트에 자격증명 직접 입력 (도구가 ENV 에서 읽도록)
- ❌ 보고서, learning log, 디버그 출력에 평문 자격증명 기록
- ❌ 이 폴더를 다른 Git 저장소·드라이브로 복사 (반드시 secrets 제외하고 복사)
- ❌ 팀 외부 인원과 공유

## 폴더 통째 공유 시 체크리스트

스킬 폴더를 새 멤버에게 전달하기 전에:

- [ ] `secrets/.env` 가 있다면 **삭제 또는 빈 템플릿으로 교체**
- [ ] `learning-log/runs/` 비우기 (민감 데이터 포함 가능)
- [ ] `.cache/`, `downloads/`, `fetched-reports/` 비우기
- [ ] 압축 후 안전한 채널로 전달
- [ ] 자격증명은 별도 채널 (1Password 등) 로 전달
