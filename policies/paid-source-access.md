# Paid Source Access Policy

> 유료 시장조사·데이터·뉴스 사이트를 합법적이고 안전하게 활용하기 위한 규칙.

---

## 기본 원칙

1. **계약·라이선스 범위 내에서만 사용**.
2. **계정 공유는 라이선스가 허용한 경우에만**.
3. **자동화 스크래핑은 사이트 정책·로봇 배제 표준 준수**.
4. **자격증명은 `secrets/.env` 에서만**, 코드·로그·프롬프트에 노출 금지.
5. **인용은 라이선스가 허용한 분량만**, 출처 명시 필수.

---

## 사이트별 라이선스 종류 (일반적)

| 라이선스 유형 | 의미 | 운영 |
|---|---|---|
| Single seat | 한 사람만 사용 가능 | 한 멤버만 자격증명 보유. 공유 시 라이선스 위반 |
| Team seat (N석) | 명시된 N명까지 동시 사용 | 자격증명 N개 또는 SSO. 멤버 변경 시 즉시 갱신 |
| Enterprise | 도메인 단위 | SSO 사용 권장. `secrets/.env` 보다 SSO 가 안전 |
| Pay-per-report | 리포트 단위 구매 | 다운로드 후 팀 내 공유 가능 여부는 리포트별 약관 확인 |
| API quota | 호출 횟수 제한 | API key 공유 가능. quota 초과 주의 |

**계약 시점**에 어떤 라이선스인지 기록하고, `references/paid-sources-registry.md` 의 "라이선스" 컬럼에 명시합니다.

---

## 절대 금지

- ❌ Single seat 라이선스를 N명이 공유 (라이선스 위반)
- ❌ 자동화 스크래핑으로 사이트 약관 우회 (사이트가 명시적으로 허용한 경우만 OK)
- ❌ 다운로드 받은 PDF·데이터를 팀 외부 또는 클라이언트에게 무단 재배포
- ❌ 인용 분량 초과 (대부분 < 100 단어 + 차트는 출처 명시 후 사용)
- ❌ "내부용" 으로 받은 자료를 외부 보고서·발표에 그대로 사용
- ❌ 사이트 robots.txt 를 우회하는 크롤링

---

## 권장 워크플로우

### 1. 사이트 도입 전

- 라이선스 약관 정독 (특히 "Permitted Use", "Redistribution", "Citation")
- 팀 라이선스인지 단일 시트인지 확인
- API 사용 가능 여부 + quota 확인
- 팀 합의 후 `references/paid-sources-registry.md` 에 등록

### 2. 자격증명 등록

- `.env.example` 에 ENV 변수 이름 추가
- 팀 공유 채널에 실제 값 등록 (1Password vault / Bitwarden / 사내 비밀 관리 도구)
- 각 멤버가 `secrets/.env` 에 채움
- 셸 로드 후 사용

### 3. 데이터 수집 (ANALYST 단계)

- 도구·스크립트가 ENV 에서 자격증명을 읽어 사이트에 인증
- 검색·다운로드 시 라이선스 허용 범위 확인
- 데이터 포인트마다 출처 (사이트명 + 리포트 ID + 날짜) 기록
- 다운로드한 원본 파일은 `downloads/` (gitignore 됨) 에 저장하거나 삭제

### 4. 보고서 인용 (WRITER·PACKAGER 단계)

- 짧은 인용 + 출처 명시 (예: "Statista, ASEAN F&B SaaS Market Report, 2025")
- 차트·표는 라이선스가 허용한 경우에만 재현. 그렇지 않으면 데이터만 추출해 자체 차트 작성
- "재구성된 데이터" 표기 권장 (예: "Source: Statista, 자체 재구성")

## 자동화 도구 사용 시 주의

### Web fetch / 브라우저 자동화

- 사이트 robots.txt 와 약관 준수
- 로그인 자동화는 라이선스가 허용한 경우만
- Rate limit 준수 (보통 초당 1-2 요청 이하)
- User-Agent 에 정직한 식별 정보 (회사명·연락처) 권장

### API 호출

- 도구가 ENV 에서 자동으로 키를 읽도록 구성
- quota 모니터링 (남은 호출 수, 갱신 시점)
- 응답 캐시 활용 (`.cache/` — gitignore)

### LLM·MCP 도구 사용 시

- LLM 에 자격증명 평문 전달 금지
- MCP 서버를 거칠 경우 환경변수가 안전하게 전달되는지 확인
- 디버그 출력에 키가 노출되지 않도록 로그 레벨 조정

## 라이선스 위반 의심 시 대응

1. **즉시 사용 중단**.
2. 무엇을 어떻게 사용했는지 기록.
3. 팀 리드·법무 (있다면) 에게 보고.
4. 사이트로부터 통지가 왔다면 정중히 응답하고 위반 시정.
5. 같은 실수를 반복하지 않도록 `policies/paid-source-access.md` 에 케이스 기록 (학습용).

---

## 무료 소스 사용 시 (참고)

DART (KR), EDINET (JP), e-Stat (JP), 통계청 KOSIS 등은 **무료지만 등록·API key 가 필요**합니다. 이들도 자격증명 정책의 적용을 받습니다 — `.env` 에 키 보관, 평문 노출 금지.

robots.txt 가 허용한 공개 페이지는 자유롭게 fetch 가능하지만, 인용 시 출처 명시는 동일하게 적용합니다.
