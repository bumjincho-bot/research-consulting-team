# Player Notation Standard — Player 표기 규칙

> ANALYST (Phase 2) 가 Player Shares 표를 작성할 때, WRITER (Phase 7) 가 보고서 본문을 쓸 때 적용.
> v1.4 KR SaaS 리서치에서 확립된 규칙.

---

## 1. 핵심 원칙

**모든 Player 에 대해 `회사명` 과 `서비스명` 을 분리 관리합니다.**

- **회사명**: DART 공시 / 법인 등기명 / 모회사명 (자회사 운영 시 모회사 명시)
- **서비스명**: 시장 인지도 높은 브랜드·앱·솔루션명

---

## 2. 표(테이블) 적용 규칙

### Player Shares 표

```markdown
| 회사명 | 서비스명 | [솔루션1] | [솔루션2] | ... | Type |
|---|---|---|---|---|---|
| 비바리퍼블리카 | 토스플레이스 | 25% | — | ... | A |
```

- **두 컬럼 의무 분리** — 단일 컬럼 "Player" 금지
- 회사·서비스 동일 시 (예: 페이히어) → 양쪽 동일 기재 (빈칸 아님)
- 자회사인 경우 → 회사명 컬럼에 모회사, 서비스명에 자회사 서비스

### 매출 Share 표

```markdown
| 회사명 | 서비스명 | 매출 (USD M) | 매출 share (%) | 출처 |
|---|---|---|---|---|
```

---

## 3. 본문 적용 규칙

- **첫 등장**: "**회사명 (서비스명)**" 풀 표기 — 예: "**비바리퍼블리카 (토스플레이스)** 는..."
- **이후**: 서비스명 short form — 예: "토스플레이스의..."
- **출처 인용**: DART 공시 인용 시 회사명 우선 — 예: "비바리퍼블리카 IR 2024"
- **약어·별칭**: 매체에서 사용하는 약어는 첫 등장 시 매핑 명시 후 사용 가능 — 예: "배달의민족 (이하 '배민')"

---

## 4. 매핑 작성 기준

### 매핑 출처 우선순위

1. **DART 공시** (사업보고서 표지 법인명 + 주요 사업 설명)
2. **금감원 전자공시 (DART)** 연결 자회사 목록
3. **회사 공식 홈페이지** IR 섹션
4. **매체 보도** (Tier C) — 1-2번으로 확인 불가 시 보조
5. **위키백과·나무위키** — 최후 수단 (검증 의무)

### 매핑 작성 형식

```markdown
| 회사명 | 서비스명 | 비고 |
|---|---|---|
| [법인명] | [시장 인지도 브랜드] | [자회사 관계·인수 이력 등] |
```

### 업데이트 주기

- **신규 리서치마다** ANALYST 가 Player 목록 점검
- 인수·분할·리브랜딩 발생 시 매핑 갱신 + 이전 명칭 비고에 보존
- learning-log 에 변경 이력 기록

---

## 5. 다국가 적용 시 주의

| 국가 | 참고 |
|---|---|
| **KR** | DART (금감원) + 법인명 정확 |
| **JP** | 有価証券報告書 (EDINET) + 법인명 (일본어 → 영문 병기) |
| **TW** | MOPS (공개資訊觀測站) + 法人名 (중문 → 영문 병기) |
| **TH** | SET (Stock Exchange of Thailand) 공시 + 법인명 (태국어 → 영문 병기) |

- 다국가 보고서에서는 **영문 회사명 (현지어 서비스명)** 형식 사용 — 예: "Woowa Brothers (배달의민족)"

---

## 6. 적용 체크리스트

- [ ] ANALYST: Player 목록 도출 시 회사-서비스 매핑 표 작성 (출처 명시)
- [ ] ANALYST: 모든 Player Shares 표에 두 컬럼 분리 적용
- [ ] WRITER: 본문 첫 등장 풀 표기 + 이후 short form 일관
- [ ] GATEKEEPER: 매핑 정확성 교차 검증 (DART 공시와 대조)
- [ ] PACKAGER: evidence log 에 매핑 출처 포함
