# Source Tiers — 소스 신뢰도 7단계

> 모든 데이터 포인트는 반드시 아래 티어 중 하나의 태그를 부착합니다. 태그가 없는 데이터는 ANALYST 가 CHECKER 로 넘기지 않습니다.

---

## 티어 정의 (공통)

| Tier | 유형 | 대표 예시 | 인용 방식 | 신뢰도 |
|---|---|---|---|---|
| **S** | 명명된 리서치 펌 리포트 | Grand View Research, Mordor Intelligence, Statista, Euromonitor, IBISWorld, Gartner, Forrester, IDC | 직접 인용 가능 | 최상 |
| **A** | 정부·국제기구·공적 통계 | 통계청 KOSIS, 일본 e-Stat, 대만 主計總處, 태국 NSO, World Bank, IMF, OECD, UN, SEC, DART, EDINET | 직접 인용 가능 | 매우 높음 |
| **B** | 기업 공식 자료 | 사업보고서, 연차보고서, 투자자 발표 자료, 공식 보도자료, 결산 공시 | 날짜 명시 인용 | 높음 |
| **C** | 공인 언론·산업 매체 | Financial Times, Bloomberg, Nikkei, 한국경제, 連結, Reuters, TechCrunch, 業界지 (industry trade press) | caveat 인용 | 중상 |
| **D** | 산업 블로그·비교 사이트 | Capterra, G2, SaaS 비교 블로그, 일반 IT 매체, 회사 자체 블로그 | 방향성 신호로만 | 중하 |
| **E** | 자체 추정 | ANALYST 가 Tier S–C 입력값으로 직접 계산한 결과 | 공식 전체 표시 필수 | 입력값 신뢰도에 따름 |
| **F** | 검증 불가 | 익명 게시물, 출처 없는 글, 위키피디아 본문, AI 생성 콘텐츠, 깨진 링크 | **사용 금지** | 없음 |

---

## 핵심 규칙

1. **F 는 절대 사용하지 않음**. 발견 시 즉시 폐기.
2. **D 또는 E 사용 시 명시적 플래그**. 보고서 본문에 "(Tier D)" 또는 "(자체 추정, Tier E)" 표기.
3. **E (자체 추정) 는 입력값과 공식을 모두 노출**. 예: "TH 시장 = 사업체 수 (NSO, Tier A) × 평균 SaaS 지출 (Statista, Tier S) × 침투율 (자체 추정 15%)".
4. **두 소스가 충돌하면 더 보수적인 값 + 범위 명시**. 평균은 평균이 적절함을 설명할 수 있을 때만.
5. **시점이 2년 이상 지난 데이터는 "older" 로 플래그**. 최신 데이터를 못 찾았다면 그 사실을 보고.

---

## 위키피디아 처리

위키피디아 **본문은 Tier F** (저자가 익명, 본문이 변동). 그러나 위키피디아 페이지 하단의 **참고문헌**을 추적해 원 출처 (Tier S~A) 에 도달했다면 그 원 출처를 인용합니다. "위키피디아 → 원 출처" 우회 경로로만 사용.

---

## 위계 충돌 처리

```
S 또는 A 와 B 가 충돌:    더 새로운 것 우선, 둘 다 명시
A 와 C 가 충돌:           A 우선. C 는 보조
S 끼리 충돌:              둘 다 명시 + 보수적 값 채택 + 충돌 사실을 caveat 로
S/A 와 D 가 충돌:         S/A 채택. D 는 폐기
D 끼리 충돌:              둘 다 폐기 또는 방향성으로만
```

## 인용 형식 (공통)

```
[데이터 값] (출처: [기관명/매체명], [리포트 제목 또는 페이지 제목], [발행일/접근일], [URL/리포트 ID], Tier [S/A/B/C/D/E])
```

### 예시

```
태국 외식업 사업체 수 약 25만 개 (출처: NSO Thailand, Business Census 2022,
2023-08-15 발행, https://nso.go.th/..., Tier A)

LMWN 의 POS 시장 점유율 약 40% (출처: Mordor Intelligence,
Thailand Restaurant Tech Market 2024, Report ID 12345, accessed 2026-05-20, Tier S)

태국 F&B SaaS 시장 규모 약 USD 80M (자체 추정, Tier E):
  사업체 수 25만 (NSO, Tier A) × SaaS 채택률 8% (Statista, Tier S)
  × 평균 연 지출 USD 400 (Mordor 추정, Tier S) = USD 80M
```

## CHECKER 가 티어 검증 시 점검할 것

- [ ] 인용된 소스가 실제로 그 티어에 해당하는가? (예: 블로그를 Tier S 로 잘못 표기 안 했는가)
- [ ] Tier S 인 경우, 리포트 ID 또는 URL 로 실제 접근 가능한가
- [ ] Tier A 인 경우, 정부·국제기구의 공식 사이트인가
- [ ] Tier E 인 경우, 입력값들이 모두 더 높은 티어인가
- [ ] Tier D, E 사용에 명시적 플래그가 있는가
- [ ] Tier F 가 섞이지 않았는가

## 티어가 애매할 때

| 케이스 | 어떻게 할까 |
|---|---|
| 정부 통계청이 발표한 보도자료 | A (정부 공식) |
| 정부 산하 연구기관의 분석 리포트 | A (institutional) |
| 대학 교수의 연구 논문 | B–C (저자·저널 신뢰도에 따라) |
| 회사 IR 자료에 실린 외부 인용 데이터 | 원 출처 추적 후 그 출처 티어로 |
| 업계 협회 발표 (예: 한국SW산업협회) | B–C (협회 신뢰도에 따라) |
| 컨설팅 펌의 무료 인사이트 글 | C (해당 회사 명성에 따라 B 가능) |
| LinkedIn 게시물 | F (저자가 명망 있어도 출처 검증 어려움) |
| 컨퍼런스 발표 자료 | B (발표자가 회사 임원이면) / C (애널리스트면) |
| AI 답변 (Perplexity, ChatGPT 등) | F. 단, AI 가 인용한 원 출처를 추적해 그 티어로 |

판단이 어려우면 **한 단계 낮은 티어**로 보수적 처리 + 플래그.
