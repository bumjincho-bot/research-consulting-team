# Metrics Standard — 1차 지표 + 보조 지표 규칙

> ANALYST (Phase 2) 가 Player Shares 표의 지표 컬럼을 작성할 때, WRITER (Phase 7) 가 표현을 통일할 때 참조.
> v1.4 KR SaaS 리서치에서 확립된 규칙.

---

## 1. 원칙

### 이중 Share (Dual Share)

모든 버티컬에서 **두 종류의 Share 표** 를 동시에 제공합니다:

| Share 유형 | 의미 | 활용 |
|---|---|---|
| **1차 지표 Share** | "이 회사가 시장 베이스를 얼마나 점유하는가" | **운영 지배력** — 사업체·가맹점·매장·의원 단위 |
| **매출 Share** | "이 회사가 시장 매출을 얼마나 가져가는가" | **수익 기여도** — USD M 단위 |

두 Share 의 차이는 **ARPU 차이** 를 드러냅니다:
- 1차 지표 share > 매출 share → 저 ARPU 고객 비중 ↑ (예: 무료·저가 가맹점)
- 1차 지표 share < 매출 share → 고 ARPU 고객 비중 ↑ (예: 대형 프랜차이즈)

## 2. 솔루션별 맞춤 표현 (1차 지표)

### 원칙: "가맹점 수" 일괄 적용 금지

각 솔루션 카테고리의 **시장 표준 용어** 를 1차 지표로 채택합니다. "가맹점 수" 는 적합한 솔루션에만 사용.

### 솔루션 유형별 표준 (KR SaaS 기준, 타국 적용 시 현지 표준으로 매핑)

| 솔루션 유형 | 1차 지표 (표준) | 보조 표기 | 보조 적용 조건 |
|---|---|---|---|
| **결제·POS** | **가맹점 수** | — | 1차가 곧 가맹점 |
| **Mobile Order (배달·주문)** | **거래액 (GMV)** | 입점 매장 수 (보조) | 양면 시장 — GMV 우위 |
| **Booking·Reservation** | **제휴 매장 수** | 활성 매장 수 (보조) | 매체 표준 |
| **Membership·Loyalty** | **B2B 가맹점 수** | 활성 매장 수 (보조) | 멤버십 SaaS 표준 |
| **EMR (의원·병원)** | **사용 의원 수** | — (보조 미적용) | 의료법상 "가맹점" 부적합 |
| **PMS (호텔)** | **도입 호텔 수** | 활성 호텔 수 (보조) | 호텔업 표준 |
| **OTA (여행 플랫폼)** | **B2B 매출** | 제휴 객실 수 (보조) | OTA 표준 |
| **e-commerce 솔루션** | **활성 쇼핑몰 수 (라이선스 수)** | — | 라이선스 발급 기준 |
| **재고·옴니채널** | **가맹점 수** | — | |
| **직원·시술 관리** | **가맹점 수** | — | |
| **마케팅 자동화** | **B2B 가맹점 수** | — | |
| **환자 CRM** | **사용 의원 수** | — | 의료법 호환 |
| **게스트 CRM·RM** | **도입 호텔 수** | 활성 호텔 수 (보조) | 호텔업 표준 |

### 보조 지표 비적용 사유 (의미 충돌)

| 솔루션 | "가맹점 수" 미적용 사유 |
|---|---|
| EMR | 의료법상 의원·병원은 "가맹점" 아님. 법적 오해 소지 |
| OTA | OTA 는 B2C 플랫폼 — "가맹점" 은 B2B 프랜차이즈 용어 |
| PMS | 호텔은 "가맹점" 아님 — "도입 호텔" 이 업계 표준 |

## 3. "활성" 정의

- **활성 매장 수**: 최근 30일 내 1회 이상 거래·로그인·예약 발생 매장
- **활성 호텔 수**: 최근 30일 내 솔루션 사용 로그가 존재하는 호텔
- **활성 쇼핑몰 수**: 최근 30일 내 1회 이상 주문 처리가 발생한 쇼핑몰

> 회사별로 "활성" 정의가 다른 경우 → footnote 로 해당 회사의 정의 명시

## 4. 표 적용 형식

### 1차 지표 Share 표 (표 헤더)

```markdown
| 회사명 | 서비스명 | [솔루션1] (1차 지표명) | [솔루션2] (1차 지표명) | ... | Type |
```

예:
```markdown
| 회사명 | 서비스명 | Mobile Order (거래액 GMV) | Payment·POS (가맹점 수) | Membership (B2B 가맹점 수) | Type |
```

### 매출 Share 표

```markdown
| 회사명 | 서비스명 | 매출 (USD M, 2024) | 매출 share (%) | 출처 |
```

### 1차 지표 표 footnote (의무)

```markdown
> **1차 지표 footnote**: [솔루션1] = [지표명] / [솔루션2] = [지표명] / ...
```

### 매출 Share 표 footnote (의무)

```markdown
> **매출 share vs 1차 지표 share 의미**: [차이 해석 1-2문장]
```

## 5. 다국가 적용

| 국가 | 주의 사항 |
|---|---|
| **KR** | 의료법 (EMR), 여신전문금융업법 (결제 가맹점) |
| **JP** | 割賦販売法 (가맹점 정의), PMS 는 "導入施設数" |
| **TW** | EMR 은 "使用院所數", 숙박은 "導入旅館數" |
| **TH** | EMR 미성숙 — "registered clinics" / OTA 는 "partner properties" |

## 6. 적용 체크리스트

- [ ] SCOPER: 각 솔루션의 1차 지표 확정 (이 표 기준)
- [ ] ANALYST: 1차 지표 + 매출 Share 두 표 모두 작성 (per vertical)
- [ ] ANALYST: 보조 가맹점 수 병기 (해당 솔루션만)
- [ ] ANALYST: 1차 지표 vs 매출 share 차이 해석 footnote 작성
- [ ] INTEGRATOR: 다국가 시 현지 표준으로 지표명 매핑 검증
- [ ] GATEKEEPER: 의미 충돌 용어 사용 여부 점검 (EMR·OTA·PMS)
